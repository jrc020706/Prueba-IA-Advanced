2. Stack de Construcción del Agente

- Agente: LangGraph (ReAct) con `create_react_agent` y `MemorySaver`.
- Base: LangChain + LangGraph para herramientas y agentes.
- Backend: FastAPI en Python.
- Frontend: HTML, CSS y JavaScript puro.
- Modelos: Groq `llama-3.1-8b-instant` y Google Gemini `models/gemini-embedding-001`.
- Voz: gTTS para TTS y Groq Whisper para transcripción.

3. Las Dos Tools Principales

- `web_search`: consulta DuckDuckGo para información actualizada de vuelos, hoteles, visas, clima y recomendaciones.
- `travel_knowledge`: consulta un índice RAG local en FAISS construido a partir de contenidos web.

Adicionales:
- `currency_converter`: convierte divisas en vivo con open.er-api.com.
- `place_image_search`: proporciona contexto visual para destinos y activa galerías de imágenes.

4. Uso de RAG sobre una web

Sí, utiliza RAG.

- Fuente: `RAG_URL`, por defecto una página de Wikipedia sobre turismo en Colombia.
- Proceso: scrape HTML -> limpieza con BeautifulSoup -> chunking con RecursiveCharacterTextSplitter -> embeddings con Google Gemini -> índice FAISS.
- Uso: `travel_knowledge` consulta FAISS antes de responder preguntas específicas de destino.

Resumen de Cumplimiento

Este proyecto cumple con los requisitos mínimos y suma puntos extra por:
- Uso de LangGraph para orquestación de agentes.
- Implementación completa del pipeline de RAG.
